# Building Modern Websites

A hands-on, project-based course covering HTML, CSS, and JavaScript — refreshed for how the web is actually built today, with AI tools woven in as part of the workflow rather than bolted on at the end.

This repo is the course. Each session is a folder. You'll clone it once in Session 0, then pull, read, code, and push as you go.

## How this repo is organized

```
building-modern-websites/
├── 00-setup/                        Start here — tools, Git, AI setup
├── session-01-web-foundations/      ...
├── session-02-html-basics/
├── ...
├── session-20-capstone-deployment/
├── capstone/                        The site you build incrementally from Session 09 onward
└── resources/                       Cheatsheets and further reading
```

Every `session-XX` folder follows the same layout:

```
session-XX-topic/
├── README.md         Explanation — concepts, syntax, examples (read this first)
├── snippets/          Small, copyable, runnable code examples for each concept
├── practical/          Guided build-along for the session
├── exercises/          Tasks for you to do on your own
└── solutions/          Reference solutions (don't peek before you try!)
```

## What you'll learn

- **HTML5 & semantic markup** — building accessible, meaningful page structure
- **CSS3** — modern selectors, Flexbox, Grid, animation, responsive design
- **JavaScript (ES6+)** — modern syntax, DOM, `fetch`, `async/await`, storage
- **APIs & browser features** — Geolocation, Canvas, Web Storage, Drag & Drop
- **AI-assisted development** — using Claude/Copilot/OpenAI effectively while coding, and calling an AI API from your own app
- **Real deployment** — shipping your project to Netlify and Vercel, with a basic CI check

## How each session works

1. Read the session's `README.md` top to bottom — it's written W3Schools-style: short explanation → syntax → example → try it yourself.
2. Copy snippets into your own scratch file and run them in the browser. Don't just read code — type it.
3. Follow the `practical/` walkthrough. From Session 9 onward, this is a feature added to `capstone/`.
4. Attempt `exercises/` before looking at `solutions/`.
5. Commit and push your work (see `00-setup/README.md` for the workflow).

## Prerequisites

- A computer with a modern browser (Chrome, Firefox, or Edge)
- No prior coding experience assumed — Session 0 sets everything up

## Where AI fits in

AI tools (Claude, ChatGPT/OpenAI, GitHub Copilot) are treated as part of a professional developer's toolkit throughout this course, not a shortcut around learning. Every session's README has a **"Where AI fits in"** section showing a *specific, honest* use of AI for that topic — including where it gets things wrong and why understanding the fundamentals still matters. Session 18 goes further and has you call an AI API directly from your own code.
